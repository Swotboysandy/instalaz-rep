# INSTALAZ

**INSTALAZ** is a no-nonsense, production‚Äëready micro app for **previewing and publishing Instagram Carousels & Reels** via the Instagram Graph API. It ships with a clean Flask dashboard, safe state tracking, and a simple `runner.py` engine that handles upload, readiness polling, and final publishing. Use it locally, in a VM, or deploy to your favorite PaaS.

> **Why this exists**: Most ‚ÄúIG autoposters‚Äù hide complexity or hard‚Äëcode assumptions. INSTALAZ is explicit, auditable, and easy to extend‚Äîperfect as a white‚Äëlabel asset for agencies and creators who need reliable scheduled or manual posting with full control.

---

## ‚ú® Features

- **Two content types**: `carousel` (multi‚Äëimage) & `reel` (video).
- **Preview before publish**: See next caption & candidate media, then choose exactly what to post.
- **One‚Äëclick manual publish** (REST endpoints provided).
- **Background run per account**: Fire & forget from the dashboard.
- **Caption rotation with persistence**: Round‚Äërobin captions from remote `.txt` files.
- **Deterministic media picking**: Image/video indices advance safely; videos tracked to avoid re‚Äëuse.
- **Per‚Äëaccount status JSON**: Human‚Äëreadable `*_status.json` with message + timestamps.
- **Clean separation**: The Flask UI (`app.py`) is thin; the engine (`runner.py`) does the heavy lifting.
- **Config‚Äëdriven**: Add/remove accounts in `accounts.json`‚Äîno code changes.
- **Environment‚Äëonly secrets**: Access tokens and IG User IDs are read from env vars, not the repo.

## üöÄ Quickstart

### 1) Create & activate a virtualenv

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
```

### 2) Install dependencies

Create a `requirements.txt` like this (or use the one in the repo):

```
Flask==3.0.3
python-dotenv==1.0.1
requests==2.32.3
```

Then install:

```bash
pip install -r requirements.txt
```

### 3) Prepare environment variables

Create a `.env` next to `app.py`. **Do not** commit real values.

```env
# Flask app
FLASK_SECRET_KEY=change-me

# Example account A (carousel)
EXAMPLE_A_TOKEN=EAAG...your_long_token...
EXAMPLE_A_IG_ID=1784...your_ig_user_id...

# Example account B (reel)
EXAMPLE_B_TOKEN=EAAG...another_token...
EXAMPLE_B_IG_ID=1784...another_user_id...
```

> You will reference these env names in `accounts.json` via `access_token_env` and `ig_user_id_env`‚Äî**never** put raw tokens/IDs inside JSON.

### 4) Create `accounts.json` (sanitized template)

```json
[
  {
    "name": "Brand A Carousel",
    "type": "carousel",
    "access_token_env": "EXAMPLE_A_TOKEN",
    "ig_user_id_env": "EXAMPLE_A_IG_ID",
    "caption_url": "https://example.com/captions.txt",
    "state_prefix": "brand_a",
    "base_url": "https://example.com/images",
    "slides_per_post": 5
  },
  {
    "name": "Brand B Reel",
    "type": "reel",
    "access_token_env": "EXAMPLE_B_TOKEN",
    "ig_user_id_env": "EXAMPLE_B_IG_ID",
    "caption_url": "https://example.com/captions.txt",
    "state_prefix": "brand_b",
    "video_base_url": "https://example.com/videos"
  }
]
```

**Fields**

- `name` ‚Äì Label for the dashboard.
- `type` ‚Äì `carousel` or `reel`.
- `access_token_env` ‚Äì Env var name that holds your IG access token.
- `ig_user_id_env` ‚Äì Env var name that holds your IG user ID.
- `caption_url` ‚Äì Public URL to a `.txt` with one caption per line.
- `state_prefix` ‚Äì Unique key for state files (`brand_b_status.json`, etc.).
- `base_url` ‚Äì (Carousel) Public base URL for images.
- `slides_per_post` ‚Äì (Carousel) Number of slides to include per post.
- `video_base_url` ‚Äì (Reel) Public base URL for `vid.mp4` & `vid (n).mp4`.

> **Image naming**: `img (1).jpg`, `img (2).jpg`, ...  
> **Alt naming (Ruthless mode)**: `ram (1).jpg`, `ram (2).jpg`, ...  
> **Videos**: `vid.mp4`, `vid (1).mp4`, `vid (2).mp4`, ...

### 5) Run the app

```bash
python app.py
# Flask will start at http://127.0.0.1:5000
```

Open the dashboard in your browser.

## üñ•Ô∏è UI & Endpoints

### Dashboard

- `GET /` ‚Äì Lists all accounts + current status
- `POST /run/<idx>` ‚Äì Start a background run for the account at index `idx`

### Status & Preview API

- `GET /status` ‚Äì Returns aggregated statuses for all accounts
- `GET /preview/<idx>` ‚Äì Returns `{ type, caption, images[] }` for carousels **or** `{ type, caption, videos[] }` for reels

### Manual Publish API

- `POST /publish/<idx>` ‚Äì Publish selected media manually

**For carousels**:
```json
{
  "images": [
    "https://example.com/images/img%20(1).jpg",
    "https://example.com/images/img%20(2).jpg",
    "https://example.com/images/img%20(3).jpg"
  ],
  "caption": "Optional custom caption"
}
```

**For reels**:
```json
{
  "video": "https://example.com/videos/vid%20(5).mp4",
  "caption": "Optional custom caption"
}
```

### Accounts UI

- `GET /account/new` ‚Äì Create new `accounts.json` entry
- `GET /account/<idx>/edit` ‚Äì Edit an existing entry

> The form dynamically shows `base_url` + `slides_per_post` for carousels, or `video_base_url` for reels.

## üß† How media & captions are chosen

- **Captions**: `runner.py` pulls your `caption_url`, splits by lines, and rotates using `{prefix}_caption.json`.
- **Carousel images**: Uses `{prefix}_image.json` to pick the next N images. Two patterns are supported:
  - `img (n).jpg` (default) if your `base_url` contains `"instimage"`
  - `ram (n).jpg` (alternate) otherwise
- **Reels**: Picks from `vid.mp4`, `vid (1).mp4`, ‚Ä¶ while avoiding ones listed in `{prefix}_video_used.json` (which is updated on publish).

## üîê Security & Privacy

- **Never commit tokens or IG IDs**: keep them only in `.env` or your platform‚Äôs secret manager.
- `accounts.json` must reference **env var names**, not raw secrets.
- Serve media (images/videos) over HTTPS for best reliability.
- Treat `*_status.json` as logs‚Äîdon‚Äôt include sensitive data there.

## üõ†Ô∏è Troubleshooting

- **`Missing environment var ...`**  
  The env var in `accounts.json` doesn‚Äôt exist in your runtime. Define it in `.env` or your PaaS.

- **`Invalid account index` (400)**  
  You called `/run/<idx>` or `/publish/<idx>` with an out‚Äëof‚Äërange index. Check `/` or `accounts.json`.

- **`Failed to fetch caption_url ...`**  
  Your caption file is not reachable or not public. Make sure it returns `200` and plain text.

- **`Media container ... failed readiness`**  
  Readiness polling timed out. Media might be invalid, too large, or the API is throttled. Try smaller files, slower cadence, or confirm account permissions.

- **`media_publish error ...`**  
  Usually indicates permissions or invalid `creation_id`. Verify your IG account is properly connected and the token has the required scopes.

- **No videos left**  
  You exhausted the available `vid*.mp4` files. Upload more or clear `{prefix}_video_used.json` if you intentionally want to repost.

**Reset this state** by deleting the files for a given `prefix`.
